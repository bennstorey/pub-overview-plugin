# Publication Overview — PDF Export + Styling

A WoodWing Studio plug-in for the **Publication Overview** application. It adds:

- **Export PDF…** (triple-dot menu, top right): merges the per-page print PDFs
  that Studio Server already stores ('output' renditions, generated by
  Studio for InDesign on layout save) into a single downloadable PDF.
  - Entire issue or a page range (`1-5, 9`)
  - Filter by workflow status and by edition
  - Pages without a print PDF fall back to their JPG page preview
  - Filename: `<Brand> - <Issue> (pages 1-5) (Complete) (North).pdf`
- **Styling** of the page grid (gear icon in the export dialog):
  - "PRESS" badge on pages whose layout is in a sent-to-press status (configurable)
  - Diagonal watermark over sent-to-press thumbnails — text and show/hide in the
    gear; **restyle the look** in the `WATERMARK_CSS` block of
    `src/70-styling-engine.js` (one clearly-marked, self-contained CSS block)
  - Status color accent bar on every page tile (official workflow status colors)
  - Deadline highlights on non-press pages — **late** (past the hard `Deadline`,
    red pulse) and **approaching** (past the soft `DeadlineSoft`, amber); each
    toggleable in the gear, and restyle the look in the `DEADLINE_CSS` block of
    `src/70-styling-engine.js`
  - Grid density control (compact / normal / large)

## Architecture

```
src/                        concatenated in filename order by build-plugin.js
├── 00-header.js            IIFE open, PoUiSdk guard, VERSION
├── 10-config.js            default settings
├── 20-studio-api.js        cookie-session JSON-RPC client + rendition download
├── 30-issue-data.js        GetPagesInfo → issue model (pages + layout states)
├── 40-rendition-source.js  RenditionSource seam: client (v1) / InDesign Server (v2 stub)
├── 50-pdf-engine.js        pdf-lib merge/embed, progress, download
├── 60-export-dialog.js     export UI + settings section
├── 70-styling-engine.js    MutationObserver tile decoration (defensive)
├── 80-settings.js          localStorage settings store
├── 90-registration.js      PoUiSdk menu action + styling lifecycle
└── 99-footer.js            IIFE close
dist/pub-overview-plugin.js GENERATED — the file registered in Studio
docs/discovery-notes.md     verified server/API/DOM findings this build rests on
```

The plug-in runs **inside the Publication Overview child application** (a
same-origin iframe at `/app/publicationoverview/`), where the `PoUiSdk` and
relayed `ContentStationSdk` globals live. See `docs/discovery-notes.md` for
the verified API shapes (GetPagesInfo, GetPages, rendition download).

## Build & deploy

```
node build-plugin.js        # writes dist/pub-overview-plugin.js
git commit && git push      # GitHub Pages redeploys in ~1 minute
```

Hosted (GitHub Pages):

```
https://bennstorey.github.io/pub-overview-plugin/dist/pub-overview-plugin.js
```

Users need a hard refresh (Cmd-Shift-R) of Studio after a deploy.

## Registration (once, Studio Management Console)

1. Management Console → Integrations → Plug-ins
2. Section **Publication Overview** (not "Studio" — this plug-in uses PoUiSdk,
   which only exists in the Publication Overview child application)
3. Add new → paste the GitHub Pages URL above → enable

## Studio Server integration notes (hard-won)

- **Auth**: cookie session + `X-WoodWing-Application: Content Station` header +
  `Ticket: null` in the JSON-RPC payload. Same pattern as the
  word-digital-article-builder plug-in.
- **Rendition downloads**: `FileUrl`s returned by `GetPages` give HTTP 400
  ("Please specify ticket param") unless `&ww-app=Content Station` is
  appended — that makes the session cookie apply.
- **`GetPagesInfo`** is the source of truth for the page grid (it is what the
  PO app itself renders from): `EditionsPages[].PageObjects[]` carries
  PageNumber/PageOrder/PageSequence/Width/Height/ParentLayoutId and
  **`OutputRenditionAvailable`** (whether a print PDF exists per page);
  `LayoutObjects[]` carries the workflow `State {Id, Name, Color}`.
- The PO UI always resolves to a **concrete edition**; "All" is not a thing
  at the GetPagesInfo level.
- The PO DOM is undocumented. All styling selectors live in the `SELECTORS`
  map in `src/70-styling-engine.js`; the engine self-disables after repeated
  DOM errors so a Studio redesign can never break the page or the export.

## Debugging

`window.__pubPdfDebug` (in the Publication Overview iframe console) exposes
version, selectors, settings, the styling engine, and `openExportDialog()`.

To test without registration: paste the built file's contents into the
iframe's console (select the `publicationoverview` context in DevTools).

## v2: high-quality export via InDesign Server

`IdsRenditionSource` in `src/40-rendition-source.js` is the seam. Contract:
`prime(layoutIds)` queues an InDesign Server export job for the selected
layouts and polls to completion; `getPagePdf(page)` resolves fresh
press-quality per-page PDFs from the job result (a job may return one
multi-page PDF per layout plus a page map — slice by the page's sequence).
Until then the settings value `engine: 'ids'` is rejected at runtime.
